2. ESTRUCTURA DE HOJAS
═══════════════════════════════════════════════════════════════════════════════════

HOJAS MAESTRAS (EDITABLES - Carga manual):
─────────────────────────────────────────────────────────────────────────────────

📋 OPERARIAS (Gray)
   Columnas: ID_OP | Nombre | Horas_Disponibles | Estado | Especialidad | Teléfono
   Uso: Caché para búsqueda rápida de nombres
   Acceso: Solo lectura → Dictionary
   Cambio: Raro (nuevo empleado, cambios de datos)

📦 REFERENCIAS (Green)
   Columnas: ID_REF | Nombre_Ref | Descripcion | Color
   Uso: Caché para búsqueda de referencias
   Acceso: Solo lectura → Dictionary
   Cambio: Poco frecuente

📐 TALLAS (Orange)
   Columnas: ID_REF | Talla | Cantidad_Total | Cantidad_Completada | Indice_Ref_Talla
   Uso: Caché filtrado (tallas por referencia)
   Acceso: Lectura → Búsqueda en caché
   Cambio: Poco frecuente

⚙️ TAREAS (Yellow)
   Columnas: ID_REF | Num_Tarea | Nombre_Tarea | Tiempo_Segundos | Dificultad | Indice_Ref_Tarea
   Uso: Caché filtrado (tareas por referencia)
   Acceso: Lectura → Búsqueda en caché
   Cambio: Poco frecuente

HOJAS OPERACIONALES (TRABAJO DIARIO):
─────────────────────────────────────────────────────────────────────────────────

📅 ASIGNACIONES_DIARIAS (Red)
   Columnas: ID_Asig | Fecha | ID_Oper | Nombre_Oper | ID_Ref | Nombre_Ref | Talla | 
             Num_Tarea | Nombre_Tarea | Cantidad | Tiempo_Unit | Tiempo_Total_Seg | 
             Tiempo_Total_Min | Estado | Unidades_Completas | Observaciones
   Uso: Almacenar asignaciones del día
   Acceso: Lectura/Escritura desde formulario
   Formato: SIN FÓRMULAS (valores calculados en VBA)
   Indexación: ID_Asig único automático

📊 RESUMEN_DIARIO (Blue)
   Columnas: Fecha | Operaria | Minutos_Asignados
   Uso: Resumen de cargas de trabajo diarias
   Acceso: Escritura automática desde mod_Gestor_Asignaciones
   Formato: Valores calculados (no fórmulas)
   Actualización: Cada vez que se crea/edita una asignación

HOJAS ADMINISTRATIVAS:
─────────────────────────────────────────────────────────────────────────────────

✅ PRODUCCION_FINALIZADA (Green)
   Uso: Registro de producciones completadas
   Acceso: Lectura de ASIGNACIONES_DIARIAS

🗂️ ARCHIVO_ASIGNACIONES (Gray)
   Uso: Archivo histórico de asignaciones antiguas
   Acceso: Migración automática de datos >6 meses

🔍 INDICES (Light Gray)
   Uso: Índices para búsquedas rápidas (opcional)
   Acceso: Lectura por caché

3. FLUJO DE DATOS
═══════════════════════════════════════════════════════════════════════════════════

FLUJO EDITAR ASIGNACIÓN:
─────────────────────────────────────────────────────────────────────────────────

1. Usuario selecciona fila en ASIGNACIONES_DIARIAS
2. Usuario hace clic en botón "✏️ EDITAR"
   └─ Macro: RegistrarFilaEnFormulario()

3. Se ejecuta:
   ├─ ObtenerFilaSeleccionada()        → FilaActual = fila elegida
   ├─ ValidarFilaAsignacion(fila)      → ¿Tiene datos válidos?
   └─ ObtenerDatosAsignacion(fila)     → datosActuales[] (Array)

4. Se abre UserForm (frmEditarAsignacion.Show)
   ├─ UserForm_Initialize()
   ├─ CargarCacheEnMemoria() si es necesario
   ├─ CargarListasOperarias()          → Dictionary → cboOperaria
   ├─ CargarListasReferencias()        → Dictionary → cboReferencia
   └─ CargarDatosParaEditar()          → datosActuales[] → Controles

5. Usuario hace cambios:
   ├─ Cambia cboOperaria
   │  └─ cboOperaria_Change()          → ObtenerNombreOperaria()
   │     └─ txtNombreOperaria = nombre automático
   │
   ├─ Cambia cboReferencia
   │  └─ cboReferencia_Change()        → ObtenerNombreReferencia()
   │     ├─ CargarTallasDeReferencia() → Filtrar Dictionary
   │     ├─ CargarTareasDeReferencia() → Filtrar Dictionary
   │     └─ cboTalla, cboNumeroTarea se llenan
   │
   ├─ Cambia cboNumeroTarea
   │  └─ cboNumeroTarea_Change()       → ObtenerNombreTarea()
   │     ├─ ObtenerTiempoUnitario()
   │     └─ RecalcularTiempos()        → tiempoTotalSeg, tiempoTotalMin
   │
   └─ Cambia txtCantidad
      └─ txtCantidad_Change()          → RecalcularTiempos()

6. Usuario hace clic en "Guardar"
   ├─ Validaciones de todos los campos
   └─ ActualizarAsignacion(fila, ...)
      ├─ ObtenerNombreReferencia()
      ├─ ObtenerNombreTarea()
      ├─ ObtenerTiempoUnitario()
      ├─ Calcular tiempoTotalSeg = cantidad * tiempoUnit
      ├─ Calcular tiempoTotalMin = tiempoTotalSeg / 60
      ├─ GuardarDatosAsignacion(fila, datos[])
      │  └─ Escribir 16 celdas en la hoja
      ├─ ActualizarResumenDiario()
      │  ├─ Acumular tiempos por operaria del día
      │  └─ Escribir en RESUMEN_DIARIO
      └─ Unload Me (cerrar formulario)

FLUJO CREAR ASIGNACIÓN:
─────────────────────────────────────────────────────────────────────────────────

1. Usuario hace clic en botón "➕ NUEVA"
   └─ Macro: CrearNuevaAsignacionUI()

2. Se ejecuta:
   ├─ ObtenerProximaFilaDisponible()   → fila = siguiente disponible
   ├─ FilaActual = fila
   ├─ FilaValida = False               → Modo creación
   └─ frmEditarAsignacion.Show

3. UserForm_Initialize() en modo creación
   ├─ Me.Caption = "✨ CREAR NUEVA ASIGNACIÓN"
   ├─ Me.BackColor = RGB(240, 250, 240) Verde claro
   ├─ CargarCacheEnMemoria()
   ├─ CargarListasOperarias()
   ├─ CargarListasReferencias()
   ├─ CargarListasEstados()
   └─ txtFecha.Value = Format(Date, "dd/mm/yyyy")

4. (Usuario llena los campos como en edición)

5. Usuario hace clic en "Guardar"
   ├─ Validaciones
   └─ CrearAsignacion(...)
      ├─ ObtenerIdAsignacion()         → ASG + Fecha + Secuencial
      ├─ Calcular tiempos
      ├─ ObtenerProximaFilaDisponible()
      ├─ Escribir 16 celdas en la hoja
      └─ ActualizarResumenDiario()

5. OPTIMIZACIONES IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════════════

⚡ VELOCIDAD:
─────────────────────────────────────────────────────────────────────────────────
• Dictionary en memoria:
  └─ Búsqueda VLOOKUP:     20-50 ms
  └─ Búsqueda Dictionary:  0.1-0.5 ms    (100-500x más rápido)

• Cálculos en VBA:
  └─ Fórmula Excel:        Evaluación en cada cambio
  └─ VBA:                  Cálculo directo (microsegundos)

• Arrays vs lectura celda a celda:
  └─ Lectura individual:   ~1-5 ms por celda × 16 = 16-80 ms
  └─ Array:                ~5-10 ms para 16 celdas (5-10x más rápido)

💾 ALMACENAMIENTO:
─────────────────────────────────────────────────────────────────────────────────
• Sin fórmulas masivas:
  └─ Archivo v2.0 (con fórmulas):   ~15-20 MB
  └─ Archivo v3.0 (sin fórmulas):   ~2-3 MB    (75-90% menos)

• Valores fijos en lugar de funciones:
  └─ Tiempo_Total_Seg = valor (no =A*B)
  └─ Reduce tiempo de guardado en 50%

🔄 CONCURRENCIA:
─────────────────────────────────────────────────────────────────────────────────
• Usuario único (actual):       ✓ Perfecto
• 2 usuarios simultáneos:       ⚠️ Posible (con compartir archivo)
• 3+ usuarios:                  ✗ Requeriría SQL Server
